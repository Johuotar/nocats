Naming the html files

Put each page in its own folder and name it index.html. If you do this, the page will be loaded automatically when you write BASE_URL/your_site/your_subpage_folder to the browser and you dont need to write BASE_URL/your_site/your_subpage_folder.subpage.html index.html is the standard for the main page that's served when you visit a domain. It works for all directories, not just domain root. http://foo.com/bar is interpreted as http://foo.com/bar/index.html on the server.

Navigation to other pages on your site & paths to resources such as images:

It is best to use relative file paths so your pages will not be bound to your current base URL. Links will work locally (localhost) and even if you move to another host. Absolute: Relative: So site navigation on a page in a folder below the home page can be done with: Homepage Subpage
