Making Website On Nekoweb Guide

This is page collects knowledge that I have learned by bashing my head against the wall until I reached enlightenment. I'm a beginner merely trying to collect useful information rather than lay down the law, so don't take these as absolute truths.


Philosophy

There are plenty of guides to basic HTML page layout and CSS to get you started, just search for any HTML + CSS tutorial, I will not waste time recreating the wheel, besides I'm not competent enough to teach you those skills. Instead I want to say this: By deciding to develop a personal website on Nekoweb, you have escaped the limitations of the standard web design. The pages you can find on Nekoweb make this quite clear with their colorful, busy even overdesigned looks. There are many moving elements that compete for your attention. This is a good thing. It means that the creators of these websites have used the opportunity to create websites that they would never be able to while working on website for business reasons. Websites that have become a rarity.

So, go and create a website with navigation circle instead of a navbar or bombard the visitor with several moving illustrations. Don't add a header and footer to your site just because you can copy them from w3schools and you don't know what else to do but copy any code you see. You will end up with a site like this if you do.

Using Nekocode

Nekocode is in-browser code editor made for creating Nekoweb sites. It has a livew preview feature that is one of the fastest ways to see how your changes affect the site. This is why it is good for beginners.

Reopening closed live preview tab

You can open live preview if you have closed it by right clicking on an active html document and choosing the first option, "Show live preview"


General web development

Reload the your edited page with CTRL + SHIFT + R

Since you visit your page regularly, your browser loads some of the site data from locally stored files. A hard refresh - CTRL + SHIFT + R - makes the browser reload all the data from the web, making your edits to the site appear. "Use it early, use it often!"

Naming the html files

Put each page in its own folder and name it index.html. If you do this, the page will be loaded automatically when you write:

base_url/your_site/your_subpage_folder

to the browser and you dont need to write:

base_url/your_site/your_subpage_folder.subpage.html

index.html is the standard for the main page that's served when you visit a domain. It works for all directories, not just domain root.

http://foo.com/bar

is interpreted as

http://foo.com/bar/index.html

on the server.

Navigation to other pages on your site & paths to resources such as images:

It is best to use relative file paths so your pages will not be bound to your current base URL. Links will work locally (localhost) and even if you move to another host.

Absolute:

<img src="https://www.website.com/images/picture.jpg" alt="picture">

Relative:

<img src="/../images/picture.jpg" alt="placeholder">

So site navigation on a page in a folder below the home page can be done with:

<a href="/../" title="back to homepage">Homepage</a>

<a href="/../another_sub_page/" title="another sub page in folder beneath home"<Subpage</a>

Add HTML to a site so it does not get interpreted, but is instead presented as code by using HTML entities

Common problems might be having greater than and less than signs be interpreted as HTML code, causing code example to turn into actual element!

HTML entities fix this. Entities look like this: &entity_name

When you want a < write & and then lt; instead.

When you want a > write & and then gt; instead.

Learn more at w3schools.

Create RSS feed

Basics of RSS feed

RSS means Really Simple Syndication. It is a way of alerting subscribers about site updates and it does so without requiring an you to collect a mailing list or any account information of any kind. I recommend using the following guide to create your RSS feeds, it's not that hard, hence the name.

Caffeine and Lasers RSS guide at Neocities

Subscribe to RSS Feeds

You can download RSS reader software to keep up to date with Nekoweb sites. You can also use them to confirm your RSS feed works. I recommend RSS Guard (GitHub) which is free & GPL-3.0 licensed.

Add RSS Icon to Your Site

Example RSS feed icon that links to your feed for RSS readers (You need to provide the icon in images folder yourself): <a href="https://yoursite.nekoweb.org/feed.xml"><img src="/images/rss_logo.png" alt="rss feed" style="width:32px; height:32px;"> Your Site Feed</a>

RSS feed and Nekoweb Features

So you have a functioning RSS feed? You're not done though. Nekoweb has unique features for RSS feeds. Learn more at: joosh.help/rss

The guide by Joosh might be outdated since I have been unable to get my RSS posts appear in Nekoweb feed when setting the link to the same webpage but with the #elementID added to the end. It might be that it is no longer possible to trick the Nekoweb feed into approving posts that use link that has already been used before, even if you add the link an element on the page to the end. If you figure out how to do that I hope you share that information.

Ensure you have added title, link, guid and pubdate to your posts in the xml file for all Nekoweb RSS features to work. Also add your RSS file to Nekoweb in the settings of your site:

https://nekoweb.org/settings/YOURSITENAMEHERE.nekoweb.org?page=rss

When you have everything above set correctly and you make rss update, your rss post should appear here here

Style your Nekoweb RSS Posts

How RSS posts appear on Nekoweb can be edited. The styling is done in elements.css, same as the sitebox.

Nekobox can be used for developing your postbox (RSS feed), just like it helps with sitebox.
