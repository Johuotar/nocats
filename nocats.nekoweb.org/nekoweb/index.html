Making Website On Nekoweb Guide

This is page collects knowledge that I have learned through the heel.


Philosophy

There are plenty of guides to basic HTML page layout and CSS to get you started, just search for any HTML + CSS tutorial, I will not waste time recreating the wheel. Instead I want to say this: By deciding to develop a personal website on Nekoweb, you have escaped the limitations of the standard web design. The pages you can find on Nekoweb make this quite clear with their colorful, busy even overdesigned looks. There are many moving elements that compete for your attention. This is a good thing. It means that the creators of these websites have used the opportunity to create websites that they would never be able to while working on website for business reasons. Websites that have become a rarity.

So, go and create a website with navigation circle instead of a navbar or bombard the visitor with several moving illustrations. Don't add a header and footer to your site just because you can copy them from w3schools and you don't know what else to do but copy any code you see. You will end up with a boring, regular website like this.

Using Nekocode

Nekocode is in-browser code editor made for creating Nekoweb sites. It has a livew preview feature that is one of the fastest ways to see how your changes affect the site. This is why it is good for beginners.

Reopening closed live preview tab

You can open live preview if you have closed it by right clicking on an active html document and choosing the first option, "Show live preview"


General web development

Naming the html files

Put each page in its own folder and name it index.html. If you do this, the page will be loaded automatically when you write:

base_url/your_site/your_subpage_folder

to the browser and you dont need to write:

base_url/your_site/your_subpage_folder.subpage.html

index.html is the standard for the main page that's served when you visit a domain. It works for all directories, not just domain root.

http://foo.com/bar

is interpreted as

http://foo.com/bar/index.html

on the server.

Navigation to other pages on your site & paths to resources such as images:

It is best to use relative file paths so your pages will not be bound to your current base URL. Links will work locally (localhost) and even if you move to another host.

Absolute:

<img src="https://www.website.com/images/picture.jpg" alt="picture">

Relative:

<img src="/../images/picture.jpg" alt="placeholder">

So site navigation on a page in a folder below the home page can be done with:

<a href="/../" title="back to homepage">Homepage</a>

<a href="/../another_sub_page/" title="another sub page in folder beneath home"<Subpage</a>
